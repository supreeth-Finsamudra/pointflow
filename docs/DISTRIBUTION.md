# Distribution runbook

Everything ships from GitHub â€” there are **no product servers**. The phone
talks to the agent directly (LAN) or via Cloudflare quick tunnels (`--tunnel`).

| What | Where | Cost |
| --- | --- | --- |
| Binaries + installers | GitHub Releases (download counts = adoption metric) | free |

## How a release works

```bash
# bump version in agent/Cargo.toml, then:
git tag v0.1.0 && git push --tags
```

`.github/workflows/release.yml` (generated by `dist`; regenerate with
`dist init` after config changes, never hand-edit) then:

1. builds the phone UI (pnpm) â€” the agent **embeds** it via rust-embed, so
   every artifact is a single self-contained binary (~4 MB);
2. builds `aarch64-apple-darwin`, `x86_64-apple-darwin`,
   `x86_64-pc-windows-msvc`;
3. publishes a GitHub Release: `.tar.xz`/`.zip` + sha256 checksums +
   `pointflow-agent-installer.sh` / `.ps1`;
4. pushes the `pointflow.rb` formula to the tap.

Users then install with any of:

```bash
brew install supreeth-Finsamudra/pointflow/pointflow                  # macOS
curl --proto '=https' --tlsv1.2 -LsSf \
  https://github.com/supreeth-Finsamudra/pointflow/releases/latest/download/pointflow-agent-installer.sh | sh
powershell -ExecutionPolicy Bypass -c "irm https://github.com/supreeth-Finsamudra/pointflow/releases/latest/download/pointflow-agent-installer.ps1 | iex"
```

## One-time setup (repo owner only â€” @supreeth-Finsamudra)

The release pipeline is ready but needs two things only the account owner can do:

1. **Create the tap repo** (public; it must have at least one commit â€”
   `actions/checkout` fails on a zero-commit repo, so seed it with a README):
   ```bash
   gh repo create supreeth-Finsamudra/homebrew-pointflow --public --add-readme \
     -d "Homebrew tap for PointFlow"
   ```
2. **Create the tap token**: a fine-grained PAT with **Contents: read/write**
   on `homebrew-pointflow`, saved as the `HOMEBREW_TAP_TOKEN` actions secret
   on the `pointflow` repo:
   ```bash
   gh secret set HOMEBREW_TAP_TOKEN -R supreeth-Finsamudra/pointflow
   ```
3. **Pick a license** â€” the repo has none, which legally means "all rights
   reserved" and blocks serious adoption (and Homebrew/winget metadata wants
   one). MIT is the conventional choice for tools like this.

## winget (after the first release exists)

winget needs the published installer URL + sha256, so it's a post-v0.1.0 step:

```bash
# from a Windows machine, easiest path:
wingetcreate new https://github.com/supreeth-Finsamudra/pointflow/releases/download/v0.1.0/pointflow-agent-x86_64-pc-windows-msvc.zip
# identifier: SupreethFinsamudra.PointFlow, then submit â€” it opens the PR
# against microsoft/winget-pkgs for you.
```

Until the PR merges, Windows users have the `.ps1` one-liner above.

## Windows hardware validation checklist (before announcing v0.1.0)

Run on a real Windows 10/11 machine (`cargo run` from a clone, or the built exe):

- [ ] Agent starts, prints QR; phone on same WiFi connects and is authed
- [ ] Trackpad: move, tap-click, double-tap, drag, two-finger scroll
      (**check scroll direction feels right** â€” flip `PX_PER_NOTCH` sign
      mapping in `input.rs` if inverted; same for horizontal)
- [ ] Typing via the text bar lands in the focused app; special keys + chords
- [ ] `>_` â†’ `+ New` spawns a shell (pwsh/powershell/cmd); xterm view streams;
      typing/quick keys work; second shell + switching works
- [ ] `claude` inside a PointFlow shell: hook events arrive as cards
      (after `pointflow-agent --install-hooks`)
- [ ] `--tunnel` with cloudflared installed: public URL works from cellular
- [ ] Keep-awake: machine doesn't sleep while agent runs
- [ ] PWA: Add to Home Screen from Android Chrome and iOS Safari

## Signing (last step â€” do when the above is green)

- **macOS**: Developer ID Application cert from the existing Apple Developer
  account; `dist` supports macOS signing+notarization via its
  `macos-sign` config â€” add credentials as repo secrets, flip the config,
  regenerate the workflow.
- **Windows**: Azure Trusted Signing (~$10/mo) once SmartScreen complaints
  appear; unsigned is acceptable for an early beta.
